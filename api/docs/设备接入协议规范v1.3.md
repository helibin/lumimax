# 设备接入协议规范 v1.3

> 状态：当前联调基线。
> 本版完全覆盖并替代旧版 `v1.2`。
> 本版已落实：移除 `meta.principalType`、移除 `attr.dietPlan.set`、新增设备端确认识别结果事件、`food.analysis.result.recognitions` 改为 `items`、`locale` 与 `market` 职责拆分。

---

## 0. 相对 v1.2 的主要升级

| 维度 | v1.2 | v1.3 | 决策来源 |
| --- | --- | --- | --- |
| 设备身份 | 隐含与用户绑定 | 匿名设备身份，`meta` 不传 `principalType` | 路线图 |
| Broker | 未明确 | EMQ X 自建为主，公有云为容灾预留 | 路线图 |
| 消息总线 | 未定义 | IoT bridge 将 MQTT 转为 RabbitMQ，再分发 biz | 路线图 |
| meal 标识 | `mealRecordId` | 仅 `mealRecordId` | 已拍板 |
| 单次回包字段 | `recognitions[]` | 改为 `items[]` | 本次调整 |
| 确认事件 | 无 | 新增 `food.analysis.confirm.request/result` | 本次调整 |
| 语言与市场 | `locale` 常被混用为业务区域 | `locale` 仅用于国际化，`market` 由设备属性决定 | 本次调整 |
| 饮食计划事件 | 可扩展 | 移除 `attr.dietPlan.set` | 本次调整 |

## 1. 设备身份与 EMQ X 接入

### 1.1 设备 `meta` 基础字段

```json
{
  "requestId": "01jvq4w0e29b41d4a716446651",
  "deviceId": "01kqaey25yg7cdqc699vr74e6r",
  "timestamp": 1710000010000,
  "event": "food.analysis.request",
  "version": "1.3",
  "locale": "en-US"
}
```

说明：

- 不再使用 `meta.principalType`。
- `locale` 仅用于多语言透传（视觉与营养调用）。
- `market` 不建议由设备侧在业务请求中自行指定；服务端优先使用设备属性中的 `market`。
- 服务端默认语言由 `DEFAULT_LOCALE` 决定，默认值为 `en-US`。

### 1.2 Topic 命名

```text
v1/connect/{deviceId}/req|res
v1/status/{deviceId}/req|res
v1/event/{deviceId}/req|res
v1/attr/{deviceId}/req|res
v1/cmd/{deviceId}/req|res
```

### 1.3 IoT bridge 与 RabbitMQ

- 设备仅与 EMQ X 交互。
- IoT bridge 负责 MQTT 事件消费与协议校验。
- IoT bridge 将业务事件转发到 RabbitMQ（统一 routing key），由 biz-service 消费。

## 5. upload.token.request / upload.url.request

### 5.1 设备上行 `upload.token.request`

`upload.token.request` 用于申请 STS 临时上传凭证。设备拿到凭证后自行使用对象存储 SDK 或兼容客户端上传到返回的 `objectKey` / `tmpPrefix` 范围内。

```json
{
  "meta": {
    "requestId": "01jvq0w5zk8m6n3p7r4t9x2yab",
    "deviceId": "01kqaey25yg7cdqc699vr74e6r",
    "timestamp": 1710000005000,
    "event": "upload.token.request",
    "version": "1.3"
  },
  "data": {
    "filename": "food-001.jpg",
    "fileType": "image/jpeg",
    "maxBytes": 5242880
  }
}
```

### 5.2 服务端下行 `upload.token.result`

```json
{
  "meta": {
    "requestId": "01jvq0w5zk8m6n3p7r4t9x2yab",
    "deviceId": "01kqaey25yg7cdqc699vr74e6r",
    "timestamp": 1710000005200,
    "event": "upload.token.result",
    "version": "1.3"
  },
  "data": {
    "code": 0,
    "msg": "ok",
    "provider": "aws",
    "mode": "credentials",
    "uploadMode": "sts_credentials",
    "bucket": "lumimax-assets",
    "region": "ap-southeast-1",
    "objectKey": "tmp-file/device/01kqaey25yg7cdqc699vr74e6r/upload_01jvq0w5zk8m6n3p7r4t9x2yab.jpg",
    "tmpPrefix": "tmp-file/device/01kqaey25yg7cdqc699vr74e6r/",
    "credentials": {
      "accessKeyId": "ASIA...",
      "secretAccessKey": "...",
      "sessionToken": "..."
    },
    "expiresAt": 1710000300000,
    "maxBytes": 5242880
  }
}
```

字段边界：

- `upload.token.result` 不返回 `uploadUrl`、`method`、`headers`，这些只属于 `upload.url.result`。
- 设备使用 `credentials` 中的临时密钥向 `bucket/region` 对应对象存储上传。
- 上传对象必须位于 `tmpPrefix` 下，推荐直接使用返回的 `objectKey`。
- 临时凭证过期时间以 `expiresAt` 为准。

### 5.3 设备上行 `upload.url.request`

`upload.url.request` 用于只申请一次性预签名上传 URL。服务端会强制按 `presigned-url` 模式创建临时对象，设备必须按下行的 `method` 和 `headers` 上传文件。

```json
{
  "meta": {
    "requestId": "01jvq0w5zk8m6n3p7r4t9x2yac",
    "deviceId": "01kqaey25yg7cdqc699vr74e6r",
    "timestamp": 1710000005000,
    "event": "upload.url.request",
    "version": "1.3"
  },
  "data": {
    "filename": "food-001.jpg",
    "fileType": "image/jpeg",
    "maxBytes": 5242880
  }
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `filename` | 否 | 原始文件名；服务端只用于生成更可读的临时对象名 |
| `fileType` | 否 | MIME 类型，如 `image/jpeg`；会转换为上传限制和 `Content-Type` |
| `maxBytes` | 否 | 最大上传字节数；不传时服务端使用默认限制 |

### 5.4 服务端下行 `upload.url.result`

```json
{
  "meta": {
    "requestId": "01jvq0w5zk8m6n3p7r4t9x2yac",
    "deviceId": "01kqaey25yg7cdqc699vr74e6r",
    "timestamp": 1710000005200,
    "event": "upload.url.result",
    "version": "1.3"
  },
  "data": {
    "code": 0,
    "msg": "ok",
    "provider": "aws",
    "uploadMode": "presigned_put",
    "bucket": "lumimax-assets",
    "region": "ap-southeast-1",
    "objectKey": "tmp-file/device/01kqaey25yg7cdqc699vr74e6r/upload_01jvq0w5zk8m6n3p7r4t9x2yac.jpg",
    "uploadUrl": "https://...signed-url...",
    "method": "PUT",
    "headers": {
      "Content-Type": "image/jpeg"
    },
    "expiresAt": 1710000300000,
    "maxBytes": 5242880
  }
}
```

设备上传要求：

- 使用 `method` 指定的方法上传，目前为 `PUT`。
- 必须携带 `headers` 中返回的所有 header，尤其是 `Content-Type`。
- 上传成功后，后续业务事件使用 `objectKey` 作为图片引用，例如 `food.analysis.request.data.imageKey`。

```json

    "requestId": "...",
    "deviceId": "...",
    "timestamp": 1710000010000,
    "event": "food.analysis.request",
    "version": "1.3",
    "locale": "en-US"
  },
  "data": {
    "mealRecordId": "01hx0000000000000000000000000002",
    "weight": 123.5,
    "unit": "g",
    "imageKey": "tmp-file/device/.../food-001.jpg",
```

规则说明：

- `meta.locale` 用于指定本次返回结果的语言。
- `data.market` 不作为常规必填字段，设备端通常不应自行指定。
- 服务端识别链路优先使用 `device.market`，其次使用既有 `mealRecord.market`，最后才回退到服务端默认 `market`。

### 7.3 `items[]` 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `foodId` | string | 食物 ID |
| `type` | string | `ingredient / prepared_dish / packaged_food / restaurant_food / mixed_meal / unknown` |
| `name` | string | 原始名称 |
| `displayName` | string | 按 `locale` 返回的展示名称 |
| `canonicalName` | string | 归一化名称 |
| `quantity` | number/null | 数量（可空） |
| `measuredWeightGram` | number/null | 实际称重克数 |
| `estimatedWeightGram` | number/null | 估算克数 |
| `confidence` | number | 0-1 |
| `source` | string | 来源，见 §7.4 |
| `provider` | string | 命中的营养 provider |
| `verifiedLevel` | string/null | `confirmed / verified / estimated / unverified` |
| `calories` | number | 当前重量预估热量 |
| `nutrition` | object | 当前重量预估营养 |
| `children` | array | 用于成品菜拆解或组合餐子项 |

顶层补充字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `estimatedNutrition` | object | 本次识别项聚合营养 |
| `requiresUserConfirmation` | boolean | 设备端是否必须进入确认/纠正流程 |
| `confirmationOptions` | array | 设备端确认选项 |
| `userCommonCandidates` | array | 用户常吃候选 |

## 13. 验收建议

- 联调通过以下链路：
  - `connect.register -> connect.register.result`
  - `upload.token.request -> upload.token.result`
  - `upload.url.request -> upload.url.result`
  - `meal.record.create -> meal.record.result`
  - `food.analysis.request -> food.analysis.result(items[])`
  - `food.analysis.confirm.request -> food.analysis.confirm.result`
  - `nutrition.analysis.request -> nutrition.analysis.result`
  - `attr.set -> attr.set.result`
  - `cmd.* -> cmd.*.result`
- 检查点：
  - `items[].source` 合法
  - `count` 可空可数值
  - 修正确认路径可写入
  - 本餐汇总 `containsEstimated` 正确
