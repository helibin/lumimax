# Lumimax 全栈交互链路设计（MVP 现状）

本文档用于前后端联调与问题排查，描述“从请求进入网关到基础服务/业务服务处理并返回”的关键链路。

## 1. 统一交互总览

```mermaid
sequenceDiagram
    participant client as 客户端(App/Admin/Device)
    participant gateway as 网关服务(gateway)
    participant base as 基础服务(base-service)
    participant biz as 业务服务(biz-service)
    participant deviceDomain as 设备子域(device)
    participant iotDomain as IoT子域(iot)
    participant dietDomain as 饮食子域(diet)
    participant db as PostgreSQL
    participant redis as Redis

    client->>gateway: HTTP /api/* 或 /api/admin/*
    gateway->>gateway: 鉴权 + 限流 + requestId 注入
    alt 基础域
        gateway->>base: gRPC 调用(auth/user/role/permission/config/storage/notification)
        base->>db: 读写基础数据
        base-->>gateway: 业务结果
    else 业务域
        gateway->>biz: gRPC 调用(device/iot/meal/food/nutrition/ota)
        alt device 域
            biz->>deviceDomain: DeviceFacade / DeviceApplication
            deviceDomain->>db: 设备/绑定/命令/遥测数据
        else iot 域
            biz->>iotDomain: IotFacade / IotApplication
            iotDomain->>db: IoT 消息与处理状态
        else diet 域
            biz->>dietDomain: DietFacade / Food / Nutrition
            dietDomain->>db: meal/item/food/recognition 数据
        end
        opt 复用基础能力
            biz->>base: gRPC (storage/config/notification...)
            base-->>biz: 结果
        end
        biz-->>gateway: 业务结果
    end
    gateway->>gateway: 统一响应包装 + 错误映射
    gateway-->>client: code/msg/data/requestId
```



## 2. 管理端核心链路

### 2.1 登录链路：`POST /api/admin/auth/login`

```mermaid
sequenceDiagram
    participant admin as 管理端页面
    participant gateway as 网关服务
    participant base as 基础服务
    participant db as PostgreSQL

    admin->>gateway: 用户名 + 密码(md5)
    gateway->>base: CallAdminSystem(auth.Login)
    base->>db: 校验 system_admins.password_hash
    alt 凭证校验通过
        base-->>gateway: 管理员资料
        gateway-->>admin: accessToken + 用户资料
    else 凭证校验失败
        base-->>gateway: UnauthorizedException
        gateway-->>admin: auth.invalid_credentials
    end
```



### 2.2 看板链路：`GET /api/admin/dashboard/overview`

```mermaid
sequenceDiagram
    participant admin as 管理端页面
    participant gateway as 网关服务
    participant base as 基础服务
    participant biz as 业务服务

    admin->>gateway: dashboard/overview
    par 用户与权限指标
        gateway->>base: CallAdminSystem(dashboard.GetOverview)
    and 设备与饮食指标
        gateway->>biz: BizDeviceFacade.Execute(admin.dashboard.overview)
    end
    base-->>gateway: todayNewUsers, roleTotal, permissionTotal...
    biz-->>gateway: deviceTotal, todayMealTotal, todayRecognitionTotal...
    gateway-->>admin: 合并后的看板数据
```



## 3. 业务服务内部子域拆解（当前实现）

### 3.1 设备子域（device）

- 入口：`BizDeviceFacadeService.Execute`
- 主要 operation（操作名）：
  - C 端：`devices.*`
  - 管理端：`admin.devices.*`、`admin.dashboard.overview`
- 主要数据：`devices`、`device_bindings`、`device_commands`、`device_shadows`

### 3.2 IoT 子域（iot）

- 入口：`BizIotFacadeService.IngestCloudMessage` / `CallAdminMessage`
- 主要 method（方法名）：
  - 管理端：`ListIotMessages`、`GetIotMessage`
- 主要数据：`iot_messages` 及 IoT 事件处理状态

### 3.3 饮食子域（diet：餐食/识别/营养）

- 入口：`BizDietFacadeService.Execute` / `CallAdmin`
- 主要 operation（操作名）：
  - C 端：`meals.create`、`meals.items.analyze`、`meals.items.confirm`、`meals.finish`、`meals.list`、`meals.get`、`foods.suggest`
  - 管理端：`meals.*`、`foods.*`、`recognitionLogs.*`
- 主要数据：`meal_records`、`meal_items`、`foods`、`food_nutritions`、`recognition_logs`

## 4. C 端餐食链路（diet 细化）

### 4.1 新建餐次与识别食物

```mermaid
sequenceDiagram
    participant app as App端
    participant gateway as 网关服务
    participant biz as 业务服务
    participant base as 基础服务
    participant db as PostgreSQL

    app->>gateway: POST /api/meals
    gateway->>biz: meals.create
    biz->>db: 创建 meal_record
    biz-->>gateway: mealRecordId
    gateway-->>app: mealRecordId

    app->>gateway: POST /api/meals/:id/items/analyze
    gateway->>biz: meals.items.analyze
    biz->>base: storage 校验/确认对象(如需)
    base-->>biz: object 校验结果
    biz->>db: 保存 meal_item + recognition_log + 聚合营养
    biz-->>gateway: food + nutrition + mealTotal
    gateway-->>app: 统一响应
```



### 4.2 diet 端到端细化（建餐 -> 识别 -> 确认 -> 完餐）

```mermaid
sequenceDiagram
    participant app as App端
    participant gateway as 网关服务
    participant diet as 饮食子域
    participant nutrition as 营养服务
    participant analysis as 识别服务
    participant base as 基础服务
    participant db as PostgreSQL

    app->>gateway: POST /api/meals（新建餐次）
    gateway->>diet: meals.create
    diet->>db: 插入 meal_records(status=created)
    diet-->>gateway: mealRecordId
    gateway-->>app: mealRecordId

    app->>gateway: POST /api/meals/:id/items/analyze（识别食物）
    gateway->>diet: meals.items.analyze
    diet->>analysis: identifyFood(imageKey)
    analysis-->>diet: candidate foods + confidence
    diet->>nutrition: resolveNutrition(foodName, weightGram)
    nutrition-->>diet: calories/protein/fat/carbs/source
    opt 需要对象校验
        diet->>base: storage.validate/confirm
        base-->>diet: object 校验通过
    end
    diet->>db: 插入 meal_items + recognition_logs
    diet->>db: 更新 meal_records 聚合营养
    diet-->>gateway: item + nutrition + mealTotal
    gateway-->>app: 统一响应

    app->>gateway: POST /api/meals/:id/items/confirm（确认食物）
    gateway->>diet: meals.items.confirm
    diet->>nutrition: 复算确认后营养
    nutrition-->>diet: confirmed nutrition
    diet->>db: 更新 meal_items + meal_records
    diet->>db: 写入 recognition_logs(confirm)
    diet-->>gateway: confirmed item + mealTotal
    gateway-->>app: 统一响应

    app->>gateway: POST /api/meals/:id/finish（结束餐次）
    gateway->>diet: meals.finish
    diet->>db: meal_records.status=finished
    diet->>db: 查询 meal_items 汇总
    diet-->>gateway: meal summary
    gateway-->>app: meal summary
```



### 4.3 管理端 diet 查询链路




## 5. 设备与 IoT 链路（MVP）

### 5.1 设备侧上行事件

```mermaid

    participant device as 设备端
    participant cloudIoT as 云IoT
    participant biz as 业务服务
    participant db as PostgreSQL

    device->>cloudIoT: 上报 telemetry/event/command result
    cloudIoT->>biz: BizIotFacade.IngestCloudMessage
    biz->>db: 记录 iot_messages + 更新设备状态/业务数据
    biz-->>cloudIoT: 处理完成（按协议）
```



### 5.2 管理端设备操作下行

```mermaid
sequenceDiagram
    participant admin as 管理端页面
    participant gateway as 网关服务
    participant biz as 业务服务
    participant cloudIoT as 云IoT
    participant device as 设备端

    admin->>gateway: 设备命令/OTA 请求
    gateway->>biz: admin.devices.commands / admin.devices.ota.*
    biz->>cloudIoT: 发布下行命令
    cloudIoT-->>device: command topic 下发
    device-->>cloudIoT: ack/result
    cloudIoT->>biz: ingest result
    biz-->>gateway: command/ota 状态
```



## 6. 鉴权与错误处理约定

- 鉴权入口只在 `gateway`（`AuthGuard`、`AdminJwtGuard`、权限守卫）
- 内部服务返回 gRPC 包装错误，gateway 统一映射 HTTP 错误
- 前端统一依赖 `error.key` 判定语义（如 `auth.invalid_credentials`、`user.token_invalid`）
- 所有响应包含 `requestId`，用于端到端日志检索
