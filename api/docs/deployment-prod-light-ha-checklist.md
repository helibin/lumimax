# 生产环境部署清单（轻量高可用版）

## 最简单结论（先看这个）

- 项目初期小团队，生产环境采用 **轻量高可用**：`2 台应用机 + 托管数据库 HA + 托管缓存 + 队列持久化`。
- 不追求大规模平台化，先保证：可上线、可监控、可回滚、可恢复。
- 海外优先沿用 AWS 全链路；国内可切换到阿里云等同类服务，但需先完成 `biz-service` 消息接入与 `base-service` 对象存储协议映射验证。
- 第三方营养库必须在生产前确定：供应商、配额、SLA、降级策略、数据合规。

---

## 1）目标与原则

- 目标：在有限预算下提供稳定服务能力，避免单机单点风险。
- 原则：
  - 核心组件优先托管（尤其数据库）
  - 最小化运维复杂度
  - 预留扩容路径（实例加副本、队列扩容、读写分离）

---

## 2）推荐拓扑（初期）

- 应用层：2 台主机（部署 `gateway`、`base-service`、`biz-service`）
- 入口层：1 个负载均衡（`ALB`/`NLB`）
- 数据层：
  - PostgreSQL 托管 HA（主备）
  - Redis 托管（单副本或主从）
  - RabbitMQ（初期可单节点 + 持久化）
- 云 IoT 与存储：IoT + 消息队列 + 对象存储（托管）

---

## 3）资源规格建议（初期）

### 3.1 应用服务器

- 数量：2 台
- 规格：`4 vCPU / 8 GB RAM / 100~200 GB SSD`
- 系统：`Ubuntu 22.04 LTS`

### 3.2 数据库与中间件

- PostgreSQL（托管 HA）：`2~4 vCPU / 8~16 GB`
- Redis：`2 vCPU / 4~8 GB`
- RabbitMQ：`2 vCPU / 4 GB`（单节点起步，监控积压）

---

## 4）国内 / 海外部署差异（生产）

## 4.1 海外（AWS 路线，改造最小）

- 推荐资源：
  - AWS IoT Core
  - SQS
  - S3
  - RDS PostgreSQL
  - ElastiCache Redis
- 链路：IoT Core -> SQS -> `biz-service`
- 优点：与你当前实现一致，迁移与运维成本最低。

### 4.2 国内（本地云合规与网络优先）

- 推荐资源（示例）：
  - 阿里云 IoT 平台
  - MNS / 消息队列
  - OSS
  - RDS PostgreSQL
  - Redis 托管
- 必做事项：
  - 验证设备接入协议与证书机制
  - 统一消息字段结构（对齐当前 `device protocol 1.0`）
  - 统一对象存储签名 URL 接口语义（对齐 `base-service`）
  - 验证公网/专线网络延迟与可用性

---

## 5）账号、AK 与权限清单

### 5.1 账号与角色

- 测试与生产账号隔离
- 运行时角色（Runtime Role）
- CI/CD 发布角色
- 只读审计角色

### 5.2 AK/SK 与凭证策略

- 生产优先使用云角色凭证 + `STS` 临时凭证
- 禁止长期静态 AK/SK 写入代码库
- 必须静态 AK/SK 时，执行定期轮换与最小权限

### 5.3 最小权限要求（示例）

- IoT：仅允许指定策略/主题范围
- 队列：仅允许指定队列读写
- 对象存储：仅允许指定 bucket + prefix
- 监控：仅允许写入指定日志/指标

---

## 6）第三方营养库准备清单（生产）

- 明确供应商与合同条款（SLA、可用区、支持响应）
- 明确 API 配额（QPS、日调用上限、超额策略）
- 明确费用模型（按调用量/包月/阶梯计费）
- 明确数据字段规范与版本策略（变更通知机制）
- 明确数据合规（版权、来源、地域合规、个人数据边界）
- 明确容灾策略（主供应商 + 备用供应商/离线缓存）
- 明确降级策略（外部失败时返回可用兜底结果）
- 准备监控指标（成功率、超时率、平均响应时间、调用成本）

---

## 7）可用性与运维最小集

- 保留上一版本镜像，支持快速回滚
- 自动备份（数据库）+ 每月恢复演练
- 核心告警（至少）：
  - gateway 5xx 激增
  - biz-service IoT 队列消费中断
  - SQS/RabbitMQ 积压超阈值
  - DB 连接数高位
  - 磁盘空间告警
- 日志集中化与保留策略（控制日志成本）

---

## 8）成本参考（初期）

> 以下为项目初期量级估算（USD/月，不含税，按区域会波动）。

### 8.1 最低可用档

- 合计约：`$415 ~ $1,040 / 月`

### 8.2 建议轻量高可用档

- 合计约：`$910 ~ $2,160 / 月`

---

## 9）上线顺序（生产）

1. 准备云资源与角色权限（IoT、队列、对象存储、DB、Redis）。
2. 准备 `configs/production/*.env` 与密钥管理策略。
3. 部署基础组件与应用服务（双机 + 负载均衡）。
4. 执行 `pnpm db:migrate` 与必要 `db:seed`。
5. 验证核心链路（设备上报、上传 token、单图称重入库、查询）。
6. 开启告警与备份检查后再对外开放流量。
